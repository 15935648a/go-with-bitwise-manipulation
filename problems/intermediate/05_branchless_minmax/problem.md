# [INTERMEDIATE] 題目 05：無分支最小值與最大值（Branchless Min / Max）

## 題目描述

實作兩個函式 `int bit_min(int a, int b)` 與 `int bit_max(int a, int b)`，
在**不使用任何 `if/else`、三元運算子 `?:`、比較分支**的情況下，
回傳兩數中較小／較大的那個。

核心技巧：利用**算術右移（Arithmetic Right Shift）** 取得符號遮罩（Sign Mask），
再用 `&` 做條件選擇，完全消除分支。

---

## 函式簽名

```c
int bit_min(int a, int b);
int bit_max(int a, int b);
```

| a    | b    | 輸出  | 說明           |
|------|------|-------|----------------|

| a    | b    | 輸出  | 說明           |
|------|------|-------|----------------|

| `-2147483648` | `2147483647` | `2147483647` | INT_MIN / INT_MAX |

## 提示

### 核心觀察：算術右移取符號遮罩

對一個有號整數 `d` 做算術右移 31 位：

```
d  =  a - b

若 a < b → d 為負數：
  d >> 31  =  0xFFFFFFFF  （全 1，即 -1）

若 a >= b → d 為非負數：
  d >> 31  =  0x00000000  （全 0）
```

這個遮罩 `sign = d >> 31` 就是你的「無分支條件」。

### bit_min 推導

```
diff = a - b
sign = diff >> 31        // a<b 時全1，否則全0

// 若 a < b：返回 a  → b + (a - b) = a
// 若 a >= b：返回 b  → b + 0 = b
min = b + (diff & sign)
```

### 位元視覺化（a=3, b=7）

```
a    = 0000 0000 0000 0000 0000 0000 0000 0011  (3)
b    = 0000 0000 0000 0000 0000 0000 0000 0111  (7)

diff = a - b
     = 0000 0000 0000 0000 0000 0000 0000 0011
     - 0000 0000 0000 0000 0000 0000 0000 0111
     = 1111 1111 1111 1111 1111 1111 1111 1100  (-4，因為 3-7=-4)

sign = diff >> 31
     = 1111 1111 1111 1111 1111 1111 1111 1111  (算術右移，全部補 1)

diff & sign
     = 1111 1111 1111 1111 1111 1111 1111 1100
   &   1111 1111 1111 1111 1111 1111 1111 1111
     = 1111 1111 1111 1111 1111 1111 1111 1100  (-4)

min  = b + (diff & sign)
     = 7 + (-4)
     = 3  ✓
```

### 位元視覺化（a=7, b=3，反序）

```
diff = 7 - 3 = 4
     = 0000 0000 0000 0000 0000 0000 0000 0100

sign = diff >> 31
     = 0000 0000 0000 0000 0000 0000 0000 0000  (正數，算術右移填 0)

diff & sign
     = 0000 0000 ...  (全 0)

min  = b + 0 = 3  ✓
```

## 限制

| 項目 | 規定 |
|------|------|
| 禁止使用 | `if`、`else`、三元運算子 `?:` |
| 禁止使用 | `<`、`>`、`<=`、`>=` 比較運算子 |
| 允許使用 | `&`、`\|`、`^`、`~`、`<<`、`>>`、`+`、`-` |
| 注意事項 | `int` 為 32-bit signed，算術右移為實作定義行為，但所有主流編譯器（GCC/Clang/MSVC）均支援 |
| 邊界挑戰 | `a = INT_MIN, b = INT_MAX` 時，`a - b` 會溢位——想想怎麼避免？（加分題） |

---

## 實際應用場景

| 領域 | 應用 |
|------|------|
| **GPU / CUDA** | warp 內做 reduction 時需要無分支 min/max，避免 thread divergence 導致所有 warp lane 都需等待 |
| **影像處理** | Pixel clamp（限制像素值在 0–255），無分支版可被 SIMD 向量化，一次處理 16 個 pixel |
| **編譯器優化** | GCC/Clang 的 `-O2` 會自動將某些 `if` 版 min/max 轉換成 `CMOV` 指令（條件移動，無分支），此題讓你手動做同樣的事 |
| **圍棋引擎** | 搜尋樹節點評估中大量的 Alpha-Beta 剪枝需要比較分數，無分支版可減少 branch misprediction penalty |
| **網路封包排程** | 優先佇列中頻繁的 min/max 比較，branchless 版本對 CPU pipeline 更友善 |
